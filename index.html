HTML

  • The head element contains information about the webpage. This contains metadata that isn't shown to the user.
  • The body element represents the visible content shown to the user.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Inline CSS uses a style attribute with a CSS rule to style a specific element only
  • Internal style sheets contains rules that only apply to that section or all the like elements of that page. (Eg: All h1 elements color:red;)
  • External CSS style sheets is the most common way to style a webpage. Uses a seperate file which keeps all CSS rules in one place making design changes easier.
  • External CSS style sheets use a 'link' element placed in the HTML's head to link the external CSS file.

Git

  • git status: checks what branch we are currently working on
  • cd: change directory
  • code .: opens the directory in VS code
  • code ..: navigate one directory back
  • git pull origin main: checks that you have the latest version of project on local machine
  • git checkout -b branch-name: "-b" creates new branch, "checkout" move working branch to new branch. (git checkout main - moves to main branch)
  • git add -A: adds ALL changes to staging area
  • git commit -m "description of commit": "commit" captures snapshot of currently staged changes, "-m description of commit" commit message

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • A function is a block of code that is designed to perform a task when called to execute the code.
  • Being able to call functions when we need to allows us to control processes efficiently.
